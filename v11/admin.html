Content authoring

Create and edit items, then download them as individual files for the content/ folders.

Workflow: edit an item → click ⬇ file to download its <id>.json → place it in the matching content/ subfolder → run python generate_manifest.py → commit. Singletons (site, products, market-segments) have their own download buttons. The manifest is regenerated by the script, never here. Needs, use cases and case studies are linked by relations (typed graph edges), not by inline id arrays — set the "Addresses need" / "Instantiates use case" fields below rather than editing a list of ids directly.
